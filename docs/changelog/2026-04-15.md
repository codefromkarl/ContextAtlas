# ContextAtlas 更新总结（2026-04-15）

本轮更新是 04-06 以来运维能力建设的一次收口：

> 把散落的健康检查、幽灵清理、catalog 重建串成一条 git hook 驱动的自动运维闭环，从"有观测"推进到"自动修复"。

## 一、在系统发展中的位置

4 月的运维能力建设经历了三个阶段：

| 阶段 | 时间 | 内容 |
|------|------|------|
| 建观测 | 04-06 ~ 04-07 | health check 面板、queue 监控、daemon 状态、告警引擎 |
| 对齐与统一 | 04-09 ~ 04-14 | 告警阈值对齐、Hub 注册表清理、application 层统一入口 |
| 自动闭环 | 04-15 | git hook 触发 → 健康检测 → 条件修复，开发者无感 |

04-15 的核心价值不是新增了某个命令，而是把前两阶段建立的能力**串成了不需要人工干预的闭环**。

## 二、为什么要更新

前两阶段留下了两个断点：

1. **观测到修复需要人工介入**：health check 能发现问题（04-06/04-07），Hub 清理能清注册表（04-14），但开发者需要手动跑命令，容易遗忘
2. **全量 health check 在项目数增长后变慢**：04-07 建立的健康面板对所有项目逐一初始化 VectorStore，项目累积后耗时显著，不适合高频触发

此外，旧架构遗留的索引目录（无 snapshot 系统）和测试泄漏会持续产生幽灵项目，`hub:cleanup-ghost`（04-14）只清理 Hub 注册表，不清理文件系统层。

## 三、如何更新

### 1. `--quick` 健康检查：适配高频场景

`analyzeIndexHealth` 新增 `quick` 参数，分两个快速路径：

- **无显式 projectIds**：跳过全部 snapshot 扫描，只查 queue + daemon
- **有显式 projectIds**：只扫描指定项目，跳过 VectorStore 和 strategySummary

这是对 04-07 建立的健康面板的**性能适配**——原始面板设计用于"完整诊断"，但自动化场景需要"快速筛查"。

### 2. 幽灵索引目录清理：补全 04-14 的清理覆盖面

04-14 的 `hub:cleanup-ghost` 清理 Hub 注册表中的幽灵项目，但不触及文件系统。新增 `cleanupStaleIndexes()` 和 `hub:cleanup-stale-indexes` CLI 命令覆盖文件系统层：

- 扫描 baseDir 下所有项目目录
- 识别无 `current` 指针且无 `snapshots` 目录但有 `index.db` 的幽灵目录（来自旧架构遗留和测试泄漏）
- 保护有 `current` 或 `snapshots` 的活跃项目

### 3. Git Hooks：把检测和修复串成闭环

post-checkout / post-merge hooks 实现三层防护：

| 层级 | 机制 | 对接的前序能力 |
|------|------|----------------|
| 防抖 | 60s 内不重复触发 | 避免快速切换分支时的重复开销 |
| 快速筛查 | `health:check --quick` | 对接 04-07 的健康面板（适配高频模式） |
| 条件清理 | dry-run 先检查，有幽灵才删除 | 对接 04-14 的 Hub 清理 + 新增的文件系统清理 |

额外设计：24h 清理节流——重操作（目录扫描和删除）每天最多执行一次，日常切换只跑轻量检查和 catalog 重建。

### 4. 安装机制

`package.json` 的 `prepare` 脚本在 `pnpm install` 时自动部署 hooks。无 `contextatlas` CLI 的环境静默跳过。

## 四、解决了什么问题

| 问题 | 来源 | 解决方案 |
|------|------|----------|
| 健康观测到修复需要人工 | 04-06/04-07 建了观测但缺闭环 | git hook 自动触发 |
| 全量 health check 太慢 | 04-07 面板未考虑项目累积 | `--quick` 跳过重操作 |
| 幽灵目录只在注册表层清理 | 04-14 只覆盖 Hub 注册表 | `cleanup-stale-indexes` 覆盖文件系统 |
| 开发者忘记手动维护 | 人总是忘 | 自动化到 checkout/merge 触发点 |

## 五、边界、兼容性与当前状态

- `health:check` 不加 `--quick` 时行为不变（完整扫描，适合手动诊断）
- `health:full` 不受影响
- Hook 安装幂等（覆盖写入 `.git/hooks/`）
- Hook 在无 `contextatlas` CLI 的环境下静默跳过
- 新增 6 个测试（3 个 quick 模式 + 3 个 stale cleanup）

新增文件：

- `scripts/setup-hooks.sh`

新 CLI 命令：

```bash
# 快速健康检查（对接 04-07 健康面板的高频模式）
contextatlas health:check --quick --json

# 清理幽灵索引目录（补全 04-14 Hub 清理的文件系统层）
contextatlas hub:cleanup-stale-indexes --dry-run --json   # 预览
contextatlas hub:cleanup-stale-indexes --json              # 执行
```
