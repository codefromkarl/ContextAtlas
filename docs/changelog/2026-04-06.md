# ContextAtlas 更新总结（2026-04-06）

本轮更新围绕一个核心收口目标展开：

> 把 ContextAtlas 从“功能逐步堆起来的 infra 工具”，收口成一条更清晰的默认主路径、一套更可信的记忆治理机制，以及一组可运维、可发布、可度量的团队能力。

## 一、为什么要更新

这次调整的背景很明确：

- 新用户第一次接入时，默认入口和推荐路径还不够清楚
- 检索结果已经有代码、记忆和决策信息，但解释、反馈和治理链路还不够统一
- 索引、发布和监控能力虽已存在，但还没有真正收口成“可运维主路径”
- 团队层面的质量退化和改进趋势，还缺少稳定的度量入口

换句话说，这次的重点不是单点加功能，而是把主路径、治理和运维能力一起拉到可依赖状态。

## 二、这次更新的目标

这轮更新希望同时解决 4 个问题：

1. 让首次使用和默认入口更清晰
2. 让检索结果更可解释，反馈真正进入主路径
3. 让记忆沉淀从“直接写入”升级为“建议后确认”的治理模式
4. 让索引、发布和团队指标进入日常可观测、可运营状态

## 三、如何更新

本轮主要通过 4 组动作推进：

### 1. 默认主路径收口

- `contextatlas start [path]` 成为真正的默认入口
- 输出统一展示仓库、`projectId`、索引状态、当前模式和下一步命令
- 新增 `docs/guides/first-use.md`，把首次接入路径压缩到可快速上手的标准流程

### 2. 检索结果与反馈链路统一

- 结果卡片固定展示代码命中、模块记忆、决策记录、命中原因、来源层级与可信规则
- 明确暴露 freshness、conflict、confidence 等状态信号
- 反馈类型正式进入主路径，包括 `helpful`、`not-helpful`、`memory-stale`、`wrong-module`

### 3. 记忆治理机制升级

- Feature Memory 更强调“建议 -> 确认”，而不是盲目自动落库
- 相似模块会提示去重或合并
- 路径或结构明显漂移时自动标记 `needs-review`
- 增强 `memory:suggest <name>` 与复核相关状态字段

### 4. 运维、发布与团队指标收口

- `contextatlas health:check` 变成更可操作的索引健康面板
- `contextatlas index:plan [path]` / `contextatlas index:update [path]` 正式进入日常更新主路径
- `pnpm release:gate` 把 build / test / smoke 收口成统一发布门禁
- `contextatlas ops:metrics` 补上仓库和模块质量分布

## 四、解决了什么问题

这次更新主要解决了以下问题：

- **首次接入门槛过高**
  用户需要自己拼装命令链路，难以快速理解“第一步做什么、第二步看什么”。
- **结果能看见，但不够解释化**
  检索结果虽然越来越丰富，但来源、可信度和记忆状态还不够直接。
- **反馈存在，但不在主路径**
  反馈能力没有真正反哺检索、治理和后续复核。
- **记忆治理偏“能写入”，不够“可治理”**
  缺少建议、确认、复核和冲突提示等稳定机制。
- **运维和发布能力不够成体系**
  索引健康、发布门禁、团队指标分散存在，难以形成持续优化闭环。

## 五、边界、兼容性与当前状态

这次更新聚焦的是主路径和治理能力，而不是底层检索算法重写：

- 重点在入口、解释、反馈、治理和运维
- 不以改变核心召回链路为目标
- 现有命令和主流程保持兼容，只是默认路径更清晰、反馈与治理能力更明确

截至这轮收口，ContextAtlas 已经更像一条完整的“接入 -> 检索 -> 反馈 -> 治理 -> 运维”主路径，而不是若干独立功能点。

## 六、涉及文件、命令与入口

本轮涉及的主要入口包括：

- `docs/guides/first-use.md`
- `contextatlas start /path/to/repo`
- `contextatlas health:check`
- `contextatlas index:plan /path/to/repo --json`
- `contextatlas index:update /path/to/repo`
- `contextatlas ops:metrics --days 7 --stale-days 30`
- `pnpm release:gate`

## 七、验证口径

本轮收口后的关键验证口径包括：

```bash
pnpm release:gate
contextatlas index:plan /path/to/repo --json
contextatlas index:update /path/to/repo
contextatlas health:check
contextatlas ops:metrics --days 7 --stale-days 30
```

说明：

- 发布门禁覆盖 build、test 和 smoke
- 主路径命令可以直接验证索引判断、健康面和团队指标是否按新路径工作
