# ContextAtlas 更新总结（2026-04-08）

本轮更新围绕新的 embedding gateway 收口，目标是把它从“最小可用转发层”推进到“可作为稳定本地接入层使用”的状态。

这次重点不在索引链路，而在 embeddings 出口能力：

> 支持多上游 failover 的本地 gateway，继续补上缓存、并发去重、Redis backend 和更完整的运维入口。

---

## 一、本轮完成了什么

本轮 embedding gateway 已具备以下能力：

1. `contextatlas gateway:embeddings` CLI 入口
2. OpenAI-compatible `POST /v1/embeddings`
3. 多上游 JSON 配置解析
4. 加权轮询
5. 网络错误、`429`、`5xx` 自动 failover
6. provider cooldown 临时摘除
7. 可选 Bearer Token 鉴权
8. `GET /healthz` 健康探针
9. 本地内存 TTL 缓存
10. 并发相同请求合并
11. 可插拔缓存 backend
12. Redis 缓存 backend

---

## 二、网关现在是什么

gateway 的定位很明确：

- 对 ContextAtlas 自身仍然表现为单一 embeddings endpoint
- 对上游 provider 则负责做多路转发、故障切换和缓存复用

这样做的直接收益：

- ContextAtlas 侧不需要改成多 provider client
- provider 级故障切换可以收敛在本地网关层
- 相同 embedding 请求可以在网关层命中缓存
- 多个并发相同请求不会重复打上游

---

## 三、本轮新增的关键能力

### 1. 本地缓存不再只是“进程内 Map”，而是抽象成 cache store

这轮把缓存层拆成了：

- `Cache Manager`
- `Memory Cache Store`
- `Redis Cache Store`

现在服务端只依赖统一 cache manager，不再把缓存细节散落在 HTTP 主流程里。

直接收益：

- 后续可以继续扩展更多 backend
- `healthz` 可以统一暴露 cache 统计
- server 主流程更容易测试

### 2. 支持 Redis backend

新增配置：

- `EMBEDDING_GATEWAY_CACHE_BACKEND`
- `EMBEDDING_GATEWAY_REDIS_URL`
- `EMBEDDING_GATEWAY_REDIS_KEY_PREFIX`

当 backend 设为 `redis` 时：

- 响应会按 cache key 写入 Redis
- TTL 使用 `PX`
- key 会加统一前缀
- 多实例部署时可以共享缓存

这意味着 gateway 不再局限于单进程内命中。

### 3. 并发相同请求合并继续保留

缓存之外，gateway 还保留 inflight request coalescing：

- 相同 payload 同时进入
- 第一个请求负责真正访问上游
- 后续同 key 请求等待同一个 promise

响应头会标识：

- `x-contextatlas-cache: miss`
- `x-contextatlas-cache: hit`
- `x-contextatlas-cache: coalesced`

这对排查缓存命中和上游放大很有用。

## 四、现在怎么配置

### 1. 内存缓存模式

```bash
EMBEDDING_GATEWAY_CACHE_TTL_MS=60000
EMBEDDING_GATEWAY_CACHE_MAX_ENTRIES=500
EMBEDDING_GATEWAY_CACHE_BACKEND=memory
```

适合：

- 单机开发
- 本地调试
- 不需要跨实例共享缓存

### 2. Redis 缓存模式

```bash
EMBEDDING_GATEWAY_CACHE_TTL_MS=60000
EMBEDDING_GATEWAY_CACHE_BACKEND=redis
EMBEDDING_GATEWAY_REDIS_URL=redis://127.0.0.1:6379/0
EMBEDDING_GATEWAY_REDIS_KEY_PREFIX=contextatlas:gateway:embeddings:
```

适合：

- 多实例部署
- 希望跨进程复用 embedding 结果
- 希望把缓存寿命独立于 gateway 进程

## 五、CLI 现在支持什么

新增/可用的 gateway CLI 选项包括：

- `--host`
- `--port`
- `--timeout-ms`
- `--failover-cooldown-ms`
- `--cache-ttl-ms`
- `--cache-max-entries`
- `--cache-backend`
- `--redis-url`
- `--redis-key-prefix`
- `--no-coalesce-identical-requests`

示例：

```bash
contextatlas gateway:embeddings \
  --port 8787 \
  --cache-backend redis \
  --redis-url redis://127.0.0.1:6379/0
```

## 六、健康检查现在能看到什么

`GET /healthz` 现在不只看 provider，也会看 cache。

返回里会包含：

- provider 可用性快照
- 当前 cache backend
- hit / miss / writes / coalesced
- inflight 数量

这让 gateway 不再只是“能用”，而是开始具备可观测性。
