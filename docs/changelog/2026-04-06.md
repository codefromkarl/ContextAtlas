# ContextAtlas 更新总结（2026-04-06）

本次更新围绕一个目标展开：

> 把 ContextAtlas 从“功能逐步堆起来的 infra 工具”，收口成一条更清晰的主路径、一套更可信的记忆治理机制，以及一组可运维、可发布、可度量的团队能力。

---

## 一、本次更新重点

本轮主要完成了四类能力收口：

1. **默认主路径更清晰**
2. **检索结果更可解释、反馈真正闭环**
3. **索引与发布流程更可运维**
4. **团队级指标可见，便于持续优化**

---

## 二、默认主路径更新

### 1. `contextatlas start [path]` 成为真正的默认入口

现在首次接入仓库时，不再需要自己拼装命令链路。`contextatlas start` 会直接给出默认闭环：

```text
Connect Repo -> Check Index Status -> Ask -> Review Result -> Give Feedback / Save Memory
```

它会明确展示：

- 当前连接的仓库
- 对应 `projectId`
- 索引是否已就绪
- 当前是完整模式还是降级模式
- 下一步推荐执行的命令

### 2. 新增首次使用文档

新增：

- `docs/guides/first-use.md`

目标是让新用户能在 10 分钟内完成：

- 安装
- 初始化
- 建索引
- 发起第一次查询
- 做第一次反馈或记忆沉淀

---

## 三、检索结果与记忆治理更新

### 1. 结果卡片结构统一

CLI 和 MCP 主路径结果都进一步统一，结果卡片会固定展示：

- 代码命中
- 相关模块记忆
- 相关决策记录
- 为什么命中这些结果
- 来源层级与可信规则
- 下一步反馈 / 沉淀命令

### 2. 来源解释、freshness、conflict、confidence 更完整

现在结果会更明确地告诉用户：

- 这条信息来自哪里（Code / Feature Memory / Decision Record / Long-term Memory / Cross-project Hub）
- 记忆是否过期（stale）
- 是否存在冲突（conflict）
- 是否为人工确认 / 建议写入 / agent 推断结果

### 3. 反馈真正进入主路径

反馈不再只是一个“存在但不常用”的命令入口，而是真正接入主路径。

现在支持的典型反馈包括：

- `helpful`
- `not-helpful`
- `memory-stale`
- `wrong-module`

这些反馈会反哺：

- 检索排序与解释
- 记忆状态判断
- 后续建议复核的模块

### 4. 记忆治理从“直接写入”改成“建议 -> 确认”

Feature Memory 现在更强调治理，而不是盲目自动落库：

- 默认建议写入，而不是直接沉淀
- 相似模块会提示去重或合并
- 源码路径或结构明显漂移时，自动标记 `needs-review`
- 人工确认成为高质量记忆的重要门槛

新增 / 强化的相关能力包括：

- `memory:suggest <name>`
- `reviewStatus`
- `reviewReason`
- `reviewMarkedAt`

---

## 四、索引与运维能力更新

### 1. 索引健康面板上线

`contextatlas health:check` 现在不只是原始健康输出，而是更接近“索引健康面板”：

- 当前状态
- 上次成功时间
- 队列长度
- 最近失败任务
- 快照版本
- 最近一次模式（full / incremental）
- 故障恢复路径

### 2. 增量更新策略进入日常可依赖状态

新增：

- `contextatlas index:plan [path]`
- `contextatlas index:update [path]`

现在系统可以更明确地区分：

- 无需更新
- 增量更新
- 需要全量重建

并且已经支持：

- repo 文件变化触发增量索引
- embedding schema 变化触发全量重建
- AST / chunking content schema 变化触发全量重建
- 显示受影响模块记忆范围（direct / routed / broad-review）

### 3. 发布门禁与 smoke gate 完整化

新增：

- `pnpm release:gate`

现在发布前门禁会结构化地执行：

- `build`
- `test`
- `smoke`

并能明确定位失败阶段。

smoke 范围也扩展到：

- CLI 基础启动
- daemon 路径
- MCP 路径
- monitoring 命令
- 冷启动搜索降级路径

---

## 五、团队级指标更新

`contextatlas ops:metrics` 现在除了汇总指标，还会输出更完整的团队质量视图。

### 已支持的指标

- 查询成功率
- 空结果率
- 用户纠错率
- memory stale 比例
- 查询延迟
- 索引失败率

### 新增的质量分布

- 仓库质量分布（repo quality distribution）
- 模块质量分布（module quality distribution）

模块质量分布会综合：

- `reviewStatus`
- `memory-stale`
- `wrong-module`
- `not-helpful`

用于识别需要优先复核的模块记忆。

## 六、本次更新带来的直接收益

对于新用户：

- 更容易第一次跑通
- 更容易理解“下一步该做什么”

对于日常使用者：

- 结果更可解释
- 错误记忆更容易反馈和治理
- 索引异常更容易定位和恢复

对于团队协作：

- 发布门禁更清晰
- 指标更完整
- 可以更早发现“哪些仓库/模块正在退化”

## 七、适合现在马上尝试的命令

```bash
contextatlas start /path/to/repo
contextatlas health:check
contextatlas index:plan /path/to/repo --json
contextatlas index:update /path/to/repo
contextatlas ops:metrics --days 7 --stale-days 30
pnpm release:gate
```
