## 二、阶段 1-2：先把前置等待时间压下去

### 1. `index:plan` / `index:update` 不再默认做接近正式扫描的工作

现在计划阶段优先基于：

- 文件列表差异
- `mtime` / `size`
- `hash` metadata
- `vector_index_hash` 收敛状态
- schema / snapshot 状态

只有真的需要时，才读取文件内容做进一步判断。

直接收益：

- `index:plan` 更适合频繁调用
- `index:update` 的前置判断更快
- daemon 执行链路的 `executionHint` 语义保持不变

### 2. staging snapshot 复制从“先整份复制”改成“按需准备”

这部分收口到低风险版本：

- 优先延迟准备 vector artifact
- 在确实需要写入时再补齐 staging 内容
- 保留 current / staging 原子切换语义

直接收益：

- 大仓库索引时的磁盘放大降低
- `index.db` / `vectors.lance` 不再无意义复制

## 三、阶段 3-4：把问题从“能修”变成“看得见、知道怎么修”

### 3. 健康检查开始区分自动修复与人工修复

`contextatlas health:check` 现在不仅会报问题，还会把修复动作分类为：

- `Auto Fixable`
- `Manual Checks`

已统一接入的典型问题包括：

- daemon 未运行
- `chunks_fts` 覆盖不足
- 向量索引缺失
- 快照缺失
- 索引数据库损坏

新增的显式修复入口包括：

- `contextatlas fts:rebuild-chunks --project-id <id>`

### 4. 队列和 daemon 现在有了专门的任务观察面

新增命令：

- `contextatlas task:status`
- `contextatlas task:inspect <taskId>`

`health:check` 也开始直接暴露：

- `oldest queued`
- `oldest running`
- `stuck running`
- `recent failures`
- `Blocked On`

`contextatlas start` 在索引未完成时也会给出：

- 队列查看入口
- 单任务排障入口

这让“为什么还没好”从黑盒状态变成了可操作状态。

## 四、阶段 5：把“空 chunk”从模糊状态改成可治理状态

这轮修复了一个很关键的稳定性问题：

过去 AST 分片失败和“文件真的没有可索引内容”都会落成 `chunks.length === 0`，随后被统一标记为 `vector_index_hash = hash`，等于把解析失败误判成“已收敛”。

现在已经拆开：

- **真实空内容**：可以标记为已收敛
- **AST 失败 / splitter 返回空 / fallback 失败**：保留待修复状态，不直接收敛

同时扩大了 fallback split 适用语言范围，包括：

- `typescript` / `javascript`
- `shell` / `powershell`
- `yaml` / `toml` / `xml`
- `html` / `css` / `scss` / `less`
- `vue` / `svelte`
- 以及多种已有语言

直接收益：

- 多语言仓库里的检索盲区减少
- 自愈不会把 parse failure 静默吞掉
- 后续健康检查和 repair 统计更可信

## 五、阶段 6：先量化，再裁剪

### 1. 新增存储冗余分析命令

新增：

- `contextatlas storage:analyze`

它会量化：

- `files.content`
- `files_fts.content`
- `chunks_fts.content`
- LanceDB `display_code`
- LanceDB `vector_text`

并给出当前可做的低风险裁剪建议。

### 2. 先裁掉一刀低风险冗余：`vector_text`

当前在线查询链路基本不读取持久化后的 `vector_text`，它主要用于索引时生成 embedding。

因此这轮先做了最小裁剪：

- 新写入 LanceDB 的记录不再持久化 `vector_text` 正文
- 仍保留 `display_code`
- 仍保留现有在线检索和 FTS 重建行为

直接收益：

- 降低了 LanceDB 内部的重复正文存储
- 不影响现有在线查询路径

## 六、阶段 7：把优化从“感觉更快”变成“能量化”

新增：

- `contextatlas perf:benchmark`

支持两种使用方式：

- 单场景：`--size small|medium|large --scenario full|incremental|repair|noop`
- 全矩阵：`--matrix`

当前基准为离线模式，不依赖外部 embedding API，重点是让门禁和回归检测稳定可跑。

已覆盖的场景：

- `full`
- `incremental`
- `repair`
- `noop`

同时 release smoke 新增了：

- `benchmark-small-noop`

这意味着基准已经不只是实验命令，而是开始进入发布回归路径。
